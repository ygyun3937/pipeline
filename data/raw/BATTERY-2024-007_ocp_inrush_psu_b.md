# 이슈 보고서: OCP-002 inrush 전류 과전류 알람 (PSU-B)

## 기본 정보
- **이슈 ID**: BATTERY-2024-007
- **알람 코드**: OCP-002
- **발생일시**: 2024-09-22 08:15:44
- **심각도**: High
- **테스트 단계**: CC_CHARGE
- **채널/유닛**: CH-05, CH-08 (PSU-B 공유)

## 증상
- CC 충전 시작 5초 이내 OCP-002 알람 발생
- 측정 전류: 14.2A (허용 상한: 12.0A)
- PSU-B 공유 채널에서만 발생, PSU-A 채널 정상
- BATTERY-2024-002와 유사하나 PSU 모델이 상이

## 원인 분석
- PSU-B 모델(XPower-3000)의 내부 출력 캐패시터 용량이 PSU-A 대비 작음
- 출력 캐패시터 부족으로 충전 시작 시 inrush 전류 억제 능력 저하
- 두 채널 동시 시작 시 캐패시터 전압 복구 전에 2차 inrush 중첩
- PSU-B는 기존 스태거드 스타트 펌웨어 업데이트 대상에서 누락됨

## 재현 단계
1. PSU-B(CH-05, CH-08) 동시 충전 시작
2. 전류 프로브로 0~30초 구간 파형 측정
3. 두 채널의 inrush 피크 타이밍 확인
4. PSU-A 동일 조건 대비 비교

## 조치 방법
1. PSU-B 채널에 펌웨어 v2.4.1 스태거드 스타트 기능 적용
2. PSU-B 채널 간 시작 딜레이 300ms로 설정 (PSU-A 200ms보다 증가)
3. 중장기: PSU-B 전량 PSU-C 모델(캐패시터 용량 개선)로 교체 계획
4. 펌웨어 업데이트 대상 장비 전수 목록 재확인 체계 구축

## 테스트 결과
- 펌웨어 업데이트 후 동시 시작 50회 테스트: 알람 0회
- 피크 전류 최대 11.3A

## 영향 범위
- PSU-B를 사용하는 CH-05, CH-08, CH-13 채널
- 펌웨어 업데이트 누락 장비 전수 점검 프로세스 신설

## 관련 이력
- BATTERY-2024-002: PSU-A 동일 현상 (최초 발견 및 대응)
- 펌웨어 업데이트 관리대장에 PSU 모델별 적용 이력 추가
