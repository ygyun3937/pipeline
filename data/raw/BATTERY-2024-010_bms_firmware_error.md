# 이슈 보고서: BMS-FW-001 펌웨어 워치독 리셋 알람

## 기본 정보
- **이슈 ID**: BATTERY-2024-010
- **알람 코드**: BMS-FW-001
- **발생일시**: 2024-10-25 14:47:03
- **심각도**: High
- **테스트 단계**: CC_CHARGE
- **채널/유닛**: CH-01, CH-03, CH-07 (간헐적 동시 발생)

## 증상
- CC 충전 중 BMS-FW-001 알람과 함께 BMS 워치독 리셋 발생
- 리셋 후 충전 재시작되나 약 90~120분 주기로 반복 발생
- 이상 발생 채널이 매번 달라 하드웨어 문제보다 소프트웨어 문제로 의심
- 이벤트 로그에서 워치독 타임아웃(WDT) 플래그 확인

## 원인 분析
- BMS 펌웨어 v2.3.8의 CAN 버스 통신 인터럽트 핸들러 처리 시간 초과
- 다채널 동시 데이터 수집 시 CAN 메시지 큐 오버플로우 발생
- 인터럽트 처리 지연으로 메인 루프 1,000ms 워치독 타임아웃 초과
- 채널 수 증가(12 → 16채널) 후 발생 빈도 증가 (설계 용량 초과)

## 재현 단계
1. 16채널 동시 충전 상태에서 CAN 메시지 트래픽 분석
2. 워치독 리셋 직전 메시지 큐 깊이 모니터링
3. 인터럽트 핸들러 수행 시간 측정 (로직 애널라이저)
4. 채널 수를 12개로 줄여 재현 여부 확인

## 조치 방법
1. BMS 펌웨어 v2.4.0으로 긴급 업데이트
   - CAN 버스 인터럽트 핸들러 최적화 (처리 시간 60% 감소)
   - 메시지 큐 크기 확장 (64 → 256 엔트리)
   - 워치독 타임아웃 1,000ms → 2,000ms로 조정
2. 16채널 이상 운용 시 데이터 수집 주기 재설정

## 테스트 결과
- 펌웨어 업데이트 후 16채널 72시간 연속 테스트: 워치독 리셋 0회
- CAN 메시지 큐 최대 사용률 42%로 안정화

## 영향 범위
- 전체 충전기 BMS 펌웨어 일괄 업데이트 완료
- 리셋 발생 기간(10/15~10/25) 충방전 데이터 무결성 검토 필요

## 관련 이력
- BATTERY-2024-020: CAN 통신 그라운드 루프와 연관 가능성 검토
- v2.4.0 릴리즈 노트에 본 이슈 수정 사항 기재
